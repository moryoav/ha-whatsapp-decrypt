# Support

Thanks for using WhatsApp Media Processor for Home Assistant.

## Getting Help

Start with the project documentation:

- `README.md` for installation, setup, actions, examples, response data, and notes.
- `whatsapp_media_processor/README.md` for add-on options.
- `whatsapp_media_processor/DOCS.md` for internal API details.
- `whatsapp_media_processor/CHANGELOG.md` for release notes.

## Reporting Bugs

Use the bug report issue template when something is not working as expected.

Please include:

- Your WhatsApp Media Processor version.
- Your Home Assistant version.
- Your installation method.
- Your Home Assistant host type.
- The affected action or media type.
- Clear reproduction steps.
- Relevant Home Assistant, add-on, or integration logs with sensitive information removed.

Do not include OpenAI keys, WhatsApp media keys, encrypted media URLs, phone numbers, private message content, tokens, private paths, or personal Home Assistant configuration.

## Feature Requests

Use the feature request issue template for new ideas or improvements.

Please explain the Home Assistant workflow you want to improve, the media type involved, and whether the change belongs in add-on processing, integration actions, setup, diagnostics, examples, or documentation.

## Security Issues

If you believe you found a security vulnerability, do not open a public issue with exploit details. Follow `SECURITY.md`.

## Scope of Support

This project is community-maintained and may not have immediate support. Keep backups of your Home Assistant configuration before changing automations or installing development builds.
